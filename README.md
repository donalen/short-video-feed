# Short Video Feed

Лента коротких вертикальных видео с постраничным листанием (TikTok / Instagram Reels), сделанная на vanilla JavaScript. Mobile и desktop поддерживаются одной кодовой базой.

## Запуск

```bash
npm install
npm run dev
```

Другие команды:

```bash
npm run build    # прод-сборка в dist/
npm run preview  # локальный просмотр прод-сборки
npm run test     # юнит-тесты (Vitest)
```

## Стек

- Vanilla JavaScript (ES-модули), без фреймворков
- [Vite](https://vitejs.dev/) — только как дев-сервер и бандлер
- [Vitest](https://vitest.dev/) + jsdom — юнит-тесты
- Нативные Web API: `IntersectionObserver`, CSS `scroll-snap`

## Как устроено решение

Каждая карточка видео — независимая секция с `scroll-snap-align`, листание идёт нативным скроллом браузера (без JS-хендлинга скролла). Клавиатурная навигация (стрелки) — тоже нативная, через фокус на скролл-контейнере.

Логика разбита на маленькие сервисы с одной ответственностью, каждый слушает свой набор видео независимо через `observe()`/`unobserve()`:

| Модуль | Отвечает за |
|---|---|
| `services/Player.js` | play/pause активного видео, mute/unmute, индикатор воспроизведения, пауза при уходе со вкладки, дебаунс автоплея при быстром скролле, fallback на muted-автоплей если браузер заблокировал звук |
| `services/VideoLoader.js` | ленивая загрузка/выгрузка `<video src>` — в DOM одновременно «горячими» остаются только предыдущее/текущее/следующее видео (окно `rootMargin: 100%`, ровно 100dvh на карточку), с дебаунсом на выгрузку против дребезга при скролле |
| `services/VideoStatus.js` | состояния загрузки/ошибки карточки, повтор по кнопке |
| `services/VideoProgress.js` | полоса прогресса воспроизведения |
| `components/VideoCard.js` | разметка одной карточки |
| `components/icons.js` | inline SVG-иконки (play/pause, mute/unmute) — без иконных библиотек |
| `utils/loadVideoSource.js` | общая функция назначения `src` из `data-src` |

### Эффективность ресурсов браузера

- Видео вне окна prev/current/next выгружаются (`removeAttribute("src")` + `load()`), а не просто ставятся на паузу — освобождается декодер.
- Дебаунс на автоплей (`Player`) и на выгрузку (`VideoLoader`) — короткое колебание скролла туда-обратно не запускает воспроизведение и не пересоздаёт `<video>` заново.
- Воспроизведение ставится на паузу при уходе со вкладки (`visibilitychange`).
- Играет только одно видео одновременно; остальные явно паузятся.
- Ноль внешних зависимостей в проде, ноль `setInterval`/поллинга — всё событийно-ориентированно.

### Desktop / mobile

На узких экранах видео растягивается на весь экран. От `768px` лента сжимается в портретную рамку 9:16, отцентрированную по высоте вьюпорта.

## Тесты

`npm run test` — юнит-тесты на Vitest + jsdom для всех сервисов (`Player`, `VideoLoader`, `VideoStatus`, `VideoProgress`) и разметки карточки, включая дебаунс-логику (через фейковые таймеры) и fallback на muted-автоплей.

## Видео

Видео для наполнения ленты лежат в `public/videos/`.
